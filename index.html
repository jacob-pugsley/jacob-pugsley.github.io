DungeonCrawl

Description: DungeonCrawl is a multiplayer RPG game based on Gauntlet (1985). Both the server and client are written entirely in Java. The random maps were generated by a Python script that randomly generates "rooms and hallways".
Related courses: CS 447 - Computer Game Design
Technologies used: Java, Python, GIMP (for some graphics)
My involvement: I was responsible for developing the item and combat systems. Randomly generated items have unique properties and effects on those who use them and different actions based on the item type.
Other contributors:

  • Julian Keller - Graphics, AI, and characters
  • Tyler Higgins - Networking
  • Dr. Scott Wallace - Designed the underlying game engine used in this project
What I learned: This project helped me learn better development strategies for working on an intensive project with a small team. After doing it, I feel more ready to approach a career in software development. More technically, I gained a greater understanding of how to write code for systems that run in a constant input/ouput loop as opposed to systems that just run once and produce some result.



Simple FishBase
Frontend code Backend code

Description: Simple FishBase is a Python-based search engine for searching fish species from FishBase, a scientific database of fish species. The original dataset is designed for researchers and as such is a bit too complex for the needs of the average fish tank owner, so the goal of this project was to design a simpler, more accessible interface.
Related courses: CS 483 - Web Data
Technologies used: JavaScript, Python, Vue.js/Node.js, Semantic UI, Whoosh!, Flask, rFishBase (API for FishBase.org)
My involvement: I designed the front end portion using Vue, JavaScript, and Axios for retrieving and displaying search results. The search results were hosted on a separate server which I designed using Flask; I created a system to parse and execute user queries using Whoosh search indexing.
Other contributors:

  • Patrick Kent - Worked with a third-party R script (rFishBase) that scraped data directly from FishBase and stored it in a CSV file for easy indexing.
  • rOpenSci/Carl Boettiger - Developers of rFishBase, a tool for getting data from FishBase.org
What I learned: The biggest thing I took away from the project was a greater understanding of Vue, which can be a very powerful tool. Before this project, I thought I had an understanding of the framework, but it turns out there was much more to learn. I also learned more about Node and designing web-based projects the way they might be designed in the industry.



WSUV Cartographers
Code Live Demo

Description: In this project, my group designed an interactive map of the WSUV campus. The current map is just a flat PDF with building names on it; it does not allow anyone to see what resources are available within the buildings and I don't think it is helpful to new and prospective students. Our map allows users to click or tap on a building on the map to view images and detailed information about the building itself and what kind of services it offers. The map also displays information on parking lots such as their color and permit cost. I had hoped this project would become the official campus map, but ultimately that did not happen.
Related courses: English 402 - Technical Writing
Technologies used: JavaScript, jQuery, Semantic UI
My involvement: I designed all of the software parts of the project including the user interface and backend. Other group members were responsible for gathering all the images and building interfaces.
Other contributors:

  • Jenee Erickson - Images and building information
  • Eden Gilmer - Images and building information
  • Taylor Bolas - Images and building information
  • Designers of the original WSUV map, which was used as a background image for this project
What I learned: Over the course of this project I learned the basics of web development and Semantic UI. I already had a strong understanding of HTML, CSS, and JavaScript but had not really designed a web application yet. I also learned how to use jQuery for the first time, which I think is a good tool to have knowledge of despite it being outdated now. I also gained some knowledge of developing web applications that will work just as well on a smaller screen such as on a mobile device. Of course, I also learned a lot about the buildings and parking lots on my university campus which has been helpful for anyone who asks me about them.



Mini FTP server

Description: The goal of this project was to develop a small file transfer server as a culmination of everything we had learned about C and Linux. The server had to respond to commands given by the user through a client interface to upload and retrieve files; it was also required to handle multiple clients at one time.
Related courses: CS 360 - Systems Programming
Technologies used: C, more specifically client/server protocols
My involvement: This was an individual project aided by some networking code provided by the instructor.
What I learned: I learned how to use a low-level programming language to create a system that could potentially be used in real life instead of just another toy project. Through this project and the course as a whole I learned how to write good and robust code in C and to read Linux manual pages to find the information I need on specific functions. I also learned that C networking is one of the most complicated systems to set up and get working short of x86 Assembly, but that it is a good way to put one's C skills to the test.
